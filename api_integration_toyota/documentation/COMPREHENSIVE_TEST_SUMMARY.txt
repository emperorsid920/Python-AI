KEY IMPROVEMENTS IN V5 PROMPT
============================================================

1. API INTEGRATION
   - Connects to real-time availability database
   - Shows actual available slots (no more static data)
   - Handles 10-minute intervals (not just 30-min)
   - Supports 6am-7pm range (not limited to 8am-5:30pm)

2. TIME HANDLING
   - Correctly interprets morning/afternoon/evening
   - Handles "after 3pm", "before noon" requests
   - Filters past times for same-day requests
   - Uses API's day-of-week labels (no calculation)

3. CONVERSATION FLOW
   - Never re-asks what customer already said
   - Proactively offers slots (doesn't wait for "what's available")
   - Handles context switching naturally
   - Post-confirmation doesn't re-ask

4. VALIDATION & SAFETY
   - 6-item required checklist before confirming
   - Validation question for dates: "Just to confirm, that's Thursday, May 7th, right?"
   - Weekend requests automatically redirect to Monday
   - Transport question always asked

5. PRESENTATION
   - Natural time format: "6am, 2pm" not "06:00, 14:00"
   - Shows 3-5 slots at a time
   - Day-of-week + date format: "Thursday, May 7th"
   - Conversational tone throughout
